# 27. 桌面客户端与离线流程

## 1. 产品形态

采用一个 Tauri Shell，通过角色与 feature bundle 提供训练、标注、编排和一站通能力，不维护四套重复客户端。业务页面复用 Web packages；原生层仅负责受控文件选择、断点上传、本地缓存、系统托盘和本地 agent 管理。

## 2. 任务

- [ ] `DESK-001` 定义 Web/native IPC allowlist，禁止任意命令、路径和 URL。
- [ ] `DESK-002` 实现大文件校验、分片、暂停恢复、带宽限制和任务队列。
- [ ] `DESK-003` 实现本地草稿加密缓存、冲突检测、过期清理和租户切换隔离。
- [ ] `DESK-004` 管理 node/dyun agent 仅调用受认证本地 API，不直接操作厂商 SDK。
- [ ] `DESK-005` 实现 Windows/Linux 签名、自动更新、回滚和离线安装。
- [ ] `DESK-006` 测试路径穿越、符号链接、恶意文件、更新劫持和离线冲突。

完成条件：桌面壳不保存长期对象存储或控制面管理员凭据；退出租户会清除对应密钥材料。
