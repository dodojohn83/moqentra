## Summary

This PR contains the fixes from the third automated review pass over the `devin/1784507980-task-31-release-packaging` integration branch. The changes harden input validation, secret handling, path containment, concurrency/resource accounting, and deployment correctness across the Rust workspace, web app, Python SDK, and one-box scripts.

## Validation & correctness

- `crates/domain/src/export.rs`: COCO export now returns `Result` and validates asset sizes, unknown labels, bbox presence/geometry, and finite area.
- `crates/domain/src/dataset.rs`: `DatasetVersion::publish` computes the manifest digest before changing state.
- `crates/domain/src/quality.rs`: `QualityRun::evaluate` counts each annotation once per class distribution, and `FrameRange` rejects negative, non-integer, and out-of-range frames.
- `crates/domain/src/conversion.rs`: `PromotionPolicy::evaluate` rejects NaN/infinite metrics and thresholds; cache key covers `sdk_version` and `capabilities`.
- `crates/domain/src/pipeline.rs`: `PipelineSpec::validate_dag` reports missing dependency nodes; `HpoRun::report_trial` rejects non-finite metrics.
- `crates/domain/src/training.rs`: `TrainingJob::start_attempt` enforces `max_attempts`; `append_metrics` rejects non-finite values.
- `crates/domain/src/inference.rs`: `Deployment::observe_replica` rejects non-finite metric values.
- `crates/domain/src/model_registry.rs`: `ModelVersion::validate` requires non-empty digests.
- `crates/domain/src/annotation.rs`: `TaskLease::renew` rejects non-positive TTL.
- `crates/domain/src/import.rs`: `progress_transfer` uses checked arithmetic and rejects overflow.

## Security & secrets

- `crates/auth/src/secrets.rs`:
  - `SecretProvider::resolve` now requires `allowed_root` and rejects paths outside it; it also rejects symlinks at the leaf.
  - `SecretRedactor` matches case-insensitively on word boundaries.
  - `is_path_within_root` canonicalizes both `path` and `root` before the prefix check to defeat symlink-based directory escapes.
- `crates/auth/src/jwt.rs`: `HmacValidator` rejects empty secrets; `ServiceAccountValidator` rejects empty tokens.
- `crates/types/src/config.rs`: `ObjectStorageConfig` now stores `access_key_id` as `SecretString`, so it is redacted in both `Debug` and JSON serialization; `S3Config` also stores both credentials as `SecretString`.
- `crates/storage/src/pool.rs`: `ConnectionPool::acquire` now resets `app.current_tenant` on checkout to prevent RLS tenant leaks from connections returned without `clear_tenant`.
- `crates/storage/src/bin/migrate.rs`: `Cli` redacts `database_url` in `Debug`.
- `crates/http-api/src/northbound.rs`: `WebhookSubscription` redacts `secret_hmac` in `Debug` and now skips serializing it to JSON; URL validation rejects IPv6 link-local, embedded credentials, and non-HTTP schemes.
- `crates/dyun-adapter/src/dyun.rs`: `Runner::validate_sandbox` now resolves symlinks on every existing ancestor of the sandbox path to prevent directory-symlink escapes; `verify_bundle` rejects empty trusted keys.
- `crates/worker-control/src/local_executor.rs`: `LocalExecutor` validates image digest format, rejects root containers, and enforces bind-mount sources stay inside `workspace_root` by canonicalizing both the source and the workspace root to prevent symlink escapes.
- `crates/observability/src/lib.rs`: `DiagnosticBundle::is_safe` recursively verifies secret-keyed values are `[REDACTED]`.
- `python/moqentra_worker/src/moqentra_worker/sdk.py`: validates `worker_root` and default `work/input/output` directories; `save_checkpoint` is confined to `work_dir` or `output_dir`; `report_metric` rejects NaN/Infinity; `work_dir`/`input_dir`/`output_dir` overrides tolerate `None`/empty values.
- `apps/web/src/core/security.ts`: secret stripping covers `api_key`/`private_key` with separators; HTML escaping is strict.
- `apps/web/src/core/uploadManager.ts`: resumes cannot start a second concurrent loop, and returned `chunkIndex` is validated.
- `apps/web/src/core/apiClient.ts`: SSE generator cancels the reader in `finally`.
- `apps/web/package.json`: removed the broken `lint` script because ESLint v9 has no flat config and no TypeScript parser, so `npm run lint` failed on a fresh clone.

## Concurrency & resource accounting

- `crates/scheduler/src/scheduler.rs`: `ClusterTopology::find_placement` uses checked multiplication for CPU, memory, and accelerator totals and compares `accelerator_count` as `u64`.
- `crates/scheduler/src/reconciler.rs`: `Lease` TTL clamped to avoid `i64` overflow; `Reconciler::run_cycle` uses `wrapping_add` for revision increments.
- `crates/object-store/src/memory.rs`: multipart completion validates part number ranges, duplicate parts, missing parts, and etag mismatches; upload records the object key and rejects key mismatches.
- `crates/object-store/src/s3.rs`: `get_object` rejects negative S3 `content_length`.
- `crates/storage/src/idempotency.rs` / `outbox.rs`: use checked/precise size accounting.

## Deployment & operations

- `deploy/onebox/backup.sh`: passes MinIO credentials via a restricted temporary `--env-file` instead of inline `-e` arguments, mounts the backup directory into the container so `mc mirror` writes to the host filesystem, and passes `PGUSER` explicitly.
- `deploy/onebox/docker-compose.yml`: `control-plane` `OIDC_ISSUER_URL` points to the `oidc` service instead of `localhost`.
- `deploy/onebox/restore.sh`: rejects tar archives with absolute/`..` paths, rejects extracted symlinks, and verifies `pg_dump.sql` is a regular file.
- `deploy/onebox/init.sh`: sets `chmod 600` on the generated TLS key.
